# JiuYin-Helper

九阴真经助手。项目已从原 TFT 助手改造为九阴专用方向，不再保留 TFT、阵容、棋盘和对局自动化入口。

## 当前阶段

阶段 2：模板采集与识别调试。

- 前端提供九阴首页、环境检查页和设置页。
- 环境检查页可以识别疑似九阴窗口，展示窗口坐标、分辨率、DPI、管理员状态和输入后端状态。
- 支持保存九阴窗口区域截图，截图用于调试坐标、缩放和后续模板采集。
- 支持 nut-js 输入探针：前台点击、普通按键、组合键和拖拽。
- F10 用于切换阶段 1 输入验证模式，F12 用于请求当前动作后停止，避免占用游戏常用的 F1~F9。
- 模板调试页可以扫描九阴模板目录，使用当前窗口截图或指定图片进行模板匹配，并输出置信度和坐标。
- 未命中模板时可保存失败样本，方便后续复盘和重新采集。
- 诊断日志会同步输出到前端日志面板和本机 `userData/logs/jiuyin/diagnostics-YYYY-MM-DD.jsonl`。
- 后端已建立 `src/backend/games/jiuyin` 阶段 2 服务边界，业务自动化仍保持禁用。

当前不会执行团练、授业、采集、生活、日常等九阴业务自动化任务。阶段 2 只做模板采集、匹配调试和失败样本复盘；真实模板素材仍需在九阴客户端中人工采集。

## 安全边界

- 只考虑窗口可见、非最小化、前台图色识别和鼠标键盘输入。
- 不做反检测、虚拟机绕过、驱动绕过、内存、封包、注入、后台窗口绑定、关闭安全软件或加入白名单。
- 如果 `nut-js` 在九阴窗口内输入无效，将先暂停业务功能开发，转为完善输入后端抽象与可替换实现。

## 诊断日志

参考开源图色脚本和收费助手常见排障方式，当前会记录窗口、截图、输入和模板匹配的关键上下文：

- 窗口识别：候选数量、命中窗口、坐标、跳过窗口数量。
- 环境检查：DPI、分辨率、管理员权限、输入后端状态。
- 截图调试：截图路径、区域坐标、目标窗口标题。
- 输入探针：动作类型、按键、窗口坐标和执行耗时。
- 模板调试：分类、模板数量、阈值、扫描步长、命中数量、最高分和失败样本路径。
